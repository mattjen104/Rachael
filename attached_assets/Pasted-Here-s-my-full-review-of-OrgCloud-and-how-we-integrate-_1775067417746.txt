**REMEMBER: / PROPOSE: / RECALL: prefix protocol** — agents write durable memories and self-modification proposals as text prefixes. Clean and clever without a complex tool-call framework.

**Programs + Skills** — proper composable agent architecture. Skills are reusable code libraries. Programs pull in skills. Agents can emit // HARDENABLE blocks that get extracted and compiled into actual runnable skills. Self-improving system.

## The problems

**1. Replit kills your agent runtime.**
agent-runtime.ts runs on a 60s in-memory setInterval tick. Replit sleeps inactive instances. Every cold start resets the tick loop, wipes the quality tracker, resets token budget counters. Your agents don't run reliably on schedule.

**2. Memory retrieval is tag-matching, not semantic.**
getMemoryContext() scores by recency + access count + exact tag string overlap. A memory tagged ["epic", "navigation"] won't surface for "EHR login workflow". No embeddings, no vector similarity. I have Qdrant running right now doing nothing.

**3. Auth is fragile.**
localStorage.getItem("orgcloud_api_key") — cleared on browser wipe, doesn't work across devices, no session expiry.

**4. PostgreSQL is a single point of failure.**
Everything goes through one Postgres connection. No retry logic, no write-ahead queue for offline capture.

**5. Proposals table is a dead end.**
Agents emit PROPOSE: changes, they land in openclawProposals, but there's no automated path from approved → applied. Just sits there.
